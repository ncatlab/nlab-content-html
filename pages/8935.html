Idea

In intensional type theory, axiom K refers to a couple of different axioms applicable to individual types to turn them into h-sets, universes to turn them into internal types of sets, where the type reflection of every element of the universe is an h-set, as well as the type theory as a whole to turn it into an actual set-level type theory, where all types are h-sets. Under certain conditions, axiom K for universes is incompatible with the univalence axiom, and axiom K for the whole type theory is incompatible with the existence of a univalent universe.

Heuristically, the axiom asserts that each term of each identity type x= Axx =_A x (of equivalences of a term x:Ax:A) is propositionally equal to the canonical reflexivity equality proof refl A(x):x= Axrefl_A(x):x =_A x.

Statement

There are multiple notions of axiom K: for individual types, for type families and Tarski universes, for hierarchies of universes, and for the entire type theory itself.

For individual types

The following definition is adapted from theorem 7.2.1 of the HoTT book. Given a type AA, axiom K is the axiom that for all elements a:Aa:A and identifications p:a= Aap:a =_A a, there is an identification K(a,p):p= a= Aarefl A(a)K(a, p):p =_{a =_A a} \mathrm{refl}_A(a). This is the same as stating that there is a dependent function

K A:∏ a:A∏ p:a= Aap= a= Aarefl A(a)K_A: \prod_{a:A} \prod_{p:a =_A a} p =_{a =_A a} \mathrm{refl}_A(a)

Axiom K implies that AA is an h-set, and can be used in the definition of an h-set. Since this axiom K applies to types, it should probably be called the typal axiom K to distinguish it from other axioms K.

For universes and type families

Axiom K for Tarski universes says that given a Tarski universe (U,T)(U, T), the typal axiom K holds for the type reflection T(A)T(A) of every element of the universe A:UA:U. This is the same as stating that there is a dependent function

K U,T:∏ A:U∏ a:T(A)∏ p:a= T(A)ap= a= T(A)arefl T(A)(a)K_{U, T}: \prod_{A:U} \prod_{a:T(A)} \prod_{p:a =_{T(A)} a} p =_{a =_{T(A)} a} \mathrm{refl}_{T(A)}(a)

In fact, this definition of axiom K makes sense for any type AA with a type family B(x)B(x) indexed by elements a:Aa:A, and could be used in the definition of family of sets. This should probably be called the familial axiom K or the Tarski universe axiom K to distinguish it from the typal axiom K above.

For the entire type theory

In the same manner that uniqueness of identity proofs could be expressed as a rule instead of an axiom, Axiom K could also be rewritten as a rule instead of an axiom, by making the hypotheses of the axioms into premises of the rule. This makes axiom K applicable not only to universes, but to the entire type theory. In this manner, axiom K then becomes the rule

Γ⊢AtypeΓ⊢a:AΓ⊢p:a= AaΓ⊢K:p= a= Aarefl A(a) \frac{\Gamma\vdash A\; type \quad \Gamma\vdash a : A \quad \quad \Gamma \vdash p : a =_A a}{ \Gamma\vdash K : p =_{a =_A a} refl_A(a)}

This should probably be called the K rule rather than axiom K, since it is no longer an axiom.

Adding this rule to the type theory turns it into a set-level type theory.

Definitional K

There is also a version of the K rule called the definitional K rule which uses definitional equality instead of the identity type. It is given by the following rule

Γ⊢AtypeΓ⊢a:AΓ⊢p:a= AaΓ⊢p≡refl A(a):a= Aa \frac{\Gamma\vdash A\; type \quad \Gamma\vdash a : A \quad \quad \Gamma \vdash p : a =_A a}{ \Gamma\vdash p \equiv refl_A(a):a =_A a}

Similarly to the typal K rule, adding the definitional K rule to a type theory turns it into a set-level type theory. The definitional K rule holds in XTT, where it follows from the definitional UIP rule, which in turn follows from boundary separation.

Properties

Unlike its logical equivalent axiom UIP, axiom K can be endowed with computational behavior: K(A,x,P,d,refl Ax)K(A,x,P,d,refl_A x) computes to dd. This gives a way to specify a computational set-level type theory.

This sort of computational axiom K can also be implemented with, and is sufficient to imply, a general scheme of function definition by pattern-matching. This is implemented in the proof assistant Agda. (The flag --without-K alters Agda’s pattern-matching scheme to a weaker version appropriate for intensional type theory, including homotopy type theory.)
